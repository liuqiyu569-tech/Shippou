# 环境配置说明（前后端）

本文档用于快速完成本地开发环境准备。按本文步骤操作后，前后端应可正常启动与联调。

## 1. 推荐版本

- Git: 2.40+
- JDK: **21**
- Maven: 3.9+
- Node.js: 20 LTS
- npm: 10+
- MySQL: 8.0（本地安装）

## 2. 仓库关键目录

```text
backend/
  pom.xml
  src/main/
    java/com/taskmanagement/   ← Java 源码
    resources/
      application.yml              ← 主配置文件
      application-dev.yml          ← 开发环境配置（数据库连接等）
      application-prod.yml         ← 生产环境配置
      db/migration/                ← Flyway 数据库迁移脚本
frontend/
```

## 3. 后端配置说明

### 3.1 配置文件机制

项目使用 YAML 格式配置文件，采用 Spring Profile 机制区分环境：

- `application.yml` - 主配置文件，包含通用配置
- `application-dev.yml` - 开发环境配置（默认激活）
- `application-prod.yml` - 生产环境配置

**本地开发时**，需要修改 `application-dev.yml` 中的数据库密码：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/taskflow?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai
    username: root
    password: 你本地的MySQL密码  # ← 修改这里
```

> ⚠️ 修改密码后请勿提交到仓库，可使用环境变量覆盖。

### 3.2 技术栈说明

- **ORM 框架**：Spring Data JPA + Hibernate
- **数据库迁移**：Flyway（自动执行 `db/migration/` 下的 SQL 脚本）
- **无需手动建表**：首次启动时 Flyway 会自动创建所需表结构

### 3.3 pom.xml 与 Maven 说明

后端常用 Maven 命令（在 `backend/` 目录执行）：

```powershell
mvn -v                      # 检查 Maven 与 JDK 版本
mvn dependency:resolve      # 预先下载依赖
mvn clean test              # 编译并运行测试
mvn spring-boot:run         # 启动后端服务
```

## 4. 本地数据库初始化

本项目使用**本地安装的 MySQL 8.0**，不依赖 Docker。

### 4.1 创建数据库

只需创建空数据库，**无需手动执行建表脚本**（Flyway 会自动处理）：

```sql
CREATE DATABASE IF NOT EXISTS taskflow 
  CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;
```

或使用命令行：

```powershell
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS taskflow CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

### 4.2 Flyway 自动迁移

首次启动后端时，Flyway 会自动执行 `db/migration/` 目录下的迁移脚本。后续如有表结构变更，只需添加新的迁移脚本，Flyway 会自动执行。

### 4.3 确认数据库信息

默认连接信息（与 `application-dev.yml` 保持一致）：

| 项目 | 默认值 |
|------|--------|
| 地址 | 127.0.0.1:3306 |
| 数据库名 | taskflow |
| 用户名 | root |
| 密码 | 你本地设置的密码 |

## 5. 后端启动步骤

1. 拉取代码。
2. 创建 MySQL 数据库（仅首次需要）。
3. 修改数据库密码（编辑 `backend/src/main/resources/application-dev.yml`）。
4. 设置 JDK 21 环境变量。
5. 进入 `backend/` 目录，执行 `mvn -v` 确认 JDK 21 与 Maven 版本。
6. 执行 `mvn dependency:resolve` 预下载依赖（首次需要）。
7. 启动 Spring Boot：
   ```powershell
   cd backend
   mvn spring-boot:run
   ```
8. 看到 `Started TaskManagementApplication` 即表示启动成功。
9. Flyway 会自动创建数据库表，无需手动执行 SQL。

## 6. 前端启动步骤

1. 拉取代码。
2. 在 `frontend/` 目录安装依赖：
   ```powershell
   npm install
   ```
3. 启动前端：
   ```powershell
   npm run dev
   ```
4. 默认本地访问地址：
   - 前端: http://localhost:5173
   - 后端: http://localhost:8080

## 7. 联调约定

- API 前缀按文档约定。
- 后端 DTO 字段使用 camelCase（例如 `dueAt`）。
- 数据库字段使用 snake_case（例如 `due_at`）。
- **不允许**把真实密码或密钥提交到仓库。
- **不要** `git add application.properties`，这是你的本地配置。

## 8. 常见问题排查

1. **3306 端口被占用**：检查本机是否已有其他 MySQL 实例在运行。

2. **前端跨域报错**：检查 `application.yml` 中 `app.cors.allowed-origins` 是否包含前端地址。

3. **启动时数据库连接失败**：
   - 确认 MySQL 服务正在运行。
   - 检查 `application-dev.yml` 中的用户名、密码和数据库名是否正确。
   - 确认已创建 `taskflow` 数据库。

4. **Flyway 迁移失败**：
   - 如果之前手动建过表，可能与 Flyway 迁移脚本冲突。
   - 解决方案：清空数据库后重新启动，让 Flyway 从头执行迁移。

5. **JDK 版本不对**：确认 `JAVA_HOME` 指向 JDK 21。

6. **Maven 找不到 spring-boot 插件**：确保在 `backend/` 目录下执行命令。
