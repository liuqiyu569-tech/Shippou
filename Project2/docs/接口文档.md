## GET 查询个人任务列表

GET /api/v1/user/tasks

可根据scope查询个人任务/团队任务/所有

### 请求参数

|名称|位置|类型|必选|中文名|说明|
|---|---|---|---|---|---|
|scope|query|string| 否 ||scope=personal：只查个人任务|
|status|query|string| 否 ||按状态筛选，可选|
|priority|query|string| 否 ||按优先级筛选，可选|
|dueStart|query|string| 否 ||按截止时间下限筛选，可选|
|dueEnd|query|string| 否 ||按截止时间上限筛选，可选|
|page|query|integer| 否 ||分页查询：第几页|

## PUT 修改个人任务

PUT /api/v1/user/tasks/{id}

当请求体中 status 字段为 DONE 时，后端需递归检查该任务所有前置任务是否均为 DONE 状态。若存在任意未完成的前置任务（含间接前置），则拒绝修改，返回 400 错误，并在 message 中说明原因。

修改成功后写入一条日志。记录内容：
```
operation_type = UPDATE
object_type = TASK_INFO
scope_type = PERSONAL
summary = 标题由【旧标题】修改为【新标题】；优先级由【旧优先级】修改为【新优先级】；等等
```
如果不是所有字段都被修改了，那么只需要在summary中写出修改的字段。比如只修改了任务的优先级，那么summary可以是：优先级由【旧优先级】修改为【新优先级】。

> Body 请求参数

```json
{
  "title": "健身",
  "description": "夜跑",
  "status": "IN_PROGRESS",
  "priority": "HIGH",
  "dueAt": "2024-04-03T23:59:59"
}
```

### 请求参数

|名称|位置|类型|必选|中文名|说明|
|---|---|---|---|---|---|
|id|path|integer| 是 ||任务的ID|
|body|body|object| 是 ||none|
|» title|body|string| 是 ||none|
|» description|body|string| 是 ||none|
|» status|body|string| 是 ||none|
|» priority|body|string| 是 ||none|
|» dueAt|body|string| 是 ||none|

> 返回示例

> 200 Response

```json
{
    "code": 200,
    "message": "修改成功",
    "data": null
}
```

> 400 Response

```json
{
  "code": 400,
  "message": "存在未完成的前置任务，不能设置状态为已完成",
  "data": null
}
```

## DELETE 删除个人任务

DELETE /api/v1/user/tasks/{id}

删除任务时，同步删除以该任务为前置或以该任务为后继的所有依赖关系记录（即清理 task_dependencies 表中所有涉及该任务的行）。
