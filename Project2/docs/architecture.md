# QuantLab 架构说明

## 设计目标

QuantLab 的目标不是模拟完整交易所，而是复现量化研究中最重要的闭环：

1. 获取和清洗行情数据。
2. 计算可解释的候选因子。
3. 通过 RankIC 和分层收益判断因子是否有预测力。
4. 使用滚动训练模型，避免未来函数。
5. 将信号输入事件驱动回测引擎。
6. 输出收益、回撤、换手率、交易明细和实验报告。

## 模块职责

| 模块 | 说明 |
| --- | --- |
| `quantlab.data` | 数据生成、CSV 加载、缺失值与异常值处理 |
| `quantlab.research` | 因子计算、横截面标准化、IC 分析、滚动模型 |
| `quantlab.backtest` | 组合构建、调仓、撮合、费用、净值和风险指标 |
| `quantlab.reporting` | HTML 实验报告生成 |
| `quantlab.api` | FastAPI 实验接口 |

## 避免未来函数的处理

- 因子使用当前和历史行情计算。
- 训练目标使用未来收益，但预测当天只使用历史训练窗口。
- 每个预测日只训练该日之前的数据。
- 回测调仓使用当日已产生的 `research_score` 排序，不读取未来净值。

## 回测假设

- 当前版本使用日频收盘价成交。
- 手续费和滑点按成交金额比例扣除。
- 只做多，不使用杠杆。
- 通过 `max_weight_per_asset` 限制单资产权重。
- 现金和持仓逐日盯市。
